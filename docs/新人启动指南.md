# 新人启动指南

这份文档给第一次拿到项目的人用，目标是用最少步骤把项目跑起来。

## 你会启动什么

这个项目分成两部分：

- `client/`：前端页面，运行后打开浏览器操作
- `server/`：后端接口，负责接收图片并调用外部图像接口

你需要同时启动前后端。

## 环境要求

- Windows PowerShell
- Node.js `20+`
- npm
- 可用的图像接口密钥

先确认 Node.js 已安装：

```powershell
node -v
npm -v
```

## 第一步：下载项目

如果你还没有代码，执行：

```powershell
git clone <你的仓库地址>
cd ai-image-workbench
```

如果代码已经在本地，直接进入项目目录即可。

## 第二步：安装依赖

先安装后端依赖：

```powershell
cd server
npm install
```

再安装前端依赖：

```powershell
cd ..\client
npm install
```

## 第三步：配置后端密钥

在 `server/` 目录下新建 `\.env.local` 文件。

最小配置：

```env
IMAGE_API_KEY=你的密钥
PORT=3001
```

推荐配置：

```env
IMAGE_API_KEY=你的密钥
IMAGE_EDIT_API_URL=https://aibibo.com/v1/images/edits
IMAGE_MODEL=gpt-image-2
IMAGE_FIELD_NAME=image[]
IMAGE_RESPONSE_FORMAT=b64_json
PORT=3001
```

说明：

- `PORT=3001` 是当前项目使用的后端端口
- `.env.local` 已被忽略，不会提交到仓库

## 第四步：启动后端

打开一个 PowerShell 窗口：

```powershell
cd e:\myprojects\ai-image-workbench\server
npm run dev
```

看到类似输出说明成功：

```text
服务已启动: http://localhost:3001
```

你也可以在浏览器打开：

```text
http://localhost:3001/api/health
```

如果返回 JSON，说明后端正常。

## 第五步：启动前端

再打开一个新的 PowerShell 窗口：

```powershell
cd e:\myprojects\ai-image-workbench\client
npm run dev
```

默认会看到类似输出：

```text
http://localhost:5173/
```

然后在浏览器打开这个地址。

## 第六步：怎么使用

1. 上传图片
2. 框选主花型区域
3. 选择快捷功能
4. 点击“开始处理”
5. 在右侧任务列表查看结果
6. 可以看原图大图、结果图大图、对比和下载

## 常见问题

### 1. 3001 端口被占用

修改 `server/.env.local` 里的 `PORT`，然后同步调整前端代理配置。

当前前端默认连的是 `3001`，相关位置：

- `client/vite.config.ts`
- `client/src/api.ts`

### 2. 启动后上传时报密钥未配置

检查：

- `server/.env.local` 是否存在
- `IMAGE_API_KEY` 是否填写正确
- 修改后是否重启了后端

### 3. 前端页面能打开，但请求失败

检查：

- 后端是否真的运行在 `3001`
- 是否访问 `http://localhost:3001/api/health` 正常
- 前端是否重启过

### 4. 上传后任务失败

优先检查：

- 接口地址是否可用
- 密钥是否有效
- 模型名是否被目标服务支持
- 返回格式是否兼容当前后端解析逻辑
